# Exercise: Calculator with Error Handling

Implement three mathematical operations that return `Result` instead of panicking or silently producing incorrect results.

## Functions to implement

```rust
// Integer division — fails if b == 0
fn dividir(a: i32, b: i32) -> Result<i32, ErrorCalc>

// Square root — fails if n < 0
fn raiz(n: f64) -> Result<f64, ErrorCalc>

// Addition that detects i32 overflow
fn suma_segura(a: i32, b: i32) -> Result<i32, ErrorCalc>
```

## Hints

- For `dividir`: check that `b != 0` before dividing.
- For `raiz`: check that `n >= 0.0` before calling `.sqrt()`.
- For `suma_segura`: use the `checked_add` method of `i32`, which returns `Option<i32>`.

```rust
// checked_add returns None if there's overflow
let result = a.checked_add(b); // Option<i32>
```

> When the tests pass, you'll have implemented explicit error handling without panics.
