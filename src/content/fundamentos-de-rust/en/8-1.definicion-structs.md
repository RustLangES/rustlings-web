# Defining Structs

## Basic struct

```rust
struct User {
    name:   String,
    email:  String,
    age:    u32,
    active: bool,
}
```

To create an instance, specify all fields:

```rust
fn main() {
    let user = User {
        name:   String::from("Alice"),
        email:  String::from("alice@example.com"),
        age:    30,
        active: true,
    };

    println!("Name: {}", user.name);
    println!("Age:  {}", user.age);
}
```

## Access and mutability

To modify fields, the entire instance must be `mut`:

```rust
fn main() {
    let mut u = User {
        name:   String::from("Bob"),
        email:  String::from("bob@example.com"),
        age:    25,
        active: true,
    };

    u.age = 26;     // OK: u is mut
    u.active = false;
    println!("{}: {}", u.name, u.age);
}
```

You can't mark only one field as mutable — it's the entire struct or nothing.

## Field init shorthand

When the parameter name matches the field, you can omit the repetition:

```rust
fn create_user(name: String, email: String) -> User {
    User {
        name,    // instead of name: name
        email,   // instead of email: email
        age: 18,
        active: true,
    }
}
```

## Struct update syntax

Create a new instance based on another, changing only some fields:

```rust
fn main() {
    let u1 = User {
        name:   String::from("Alice"),
        email:  String::from("alice@example.com"),
        age:    30,
        active: true,
    };

    let u2 = User {
        email: String::from("alice_new@example.com"),
        ..u1   // take the rest from u1
    };

    // Note: u1.name was moved to u2, so u1 is no longer fully valid
    println!("{}", u2.name);
}
```

## Struct types

### Tuple structs

```rust
struct Color(u8, u8, u8);
struct Point3D(f64, f64, f64);

fn main() {
    let red    = Color(255, 0, 0);
    let origin = Point3D(0.0, 0.0, 0.0);

    println!("R={} G={} B={}", red.0, red.1, red.2);

    let Color(r, g, b) = red; // destructuring
    println!("r={r}");
}
```

They're useful for creating types with different semantics even when they have the same internal structure. `Color` and `Point3D` are distinct types even though both contain three values.

### Unit structs

```rust
struct Marker;

fn main() {
    let _m = Marker;
}
```

Useful for implementing traits without needing data.

## Debug with `#[derive(Debug)]`

To be able to print a struct with `{:?}`, add the derive:

```rust
#[derive(Debug)]
struct Rectangle {
    width:  u32,
    height: u32,
}

fn main() {
    let rect = Rectangle { width: 30, height: 50 };
    println!("{:?}", rect);   // Rectangle { width: 30, height: 50 }
    println!("{:#?}", rect);  // pretty-print
}
```
