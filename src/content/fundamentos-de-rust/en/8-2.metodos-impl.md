# Methods with `impl`

Methods are defined in `impl` (implementation) blocks. They are functions that have access to the struct's data through `self`.

## Instance methods

```rust
#[derive(Debug)]
struct Circle {
    radius: f64,
}

impl Circle {
    fn area(&self) -> f64 {
        std::f64::consts::PI * self.radius * self.radius
    }

    fn perimeter(&self) -> f64 {
        2.0 * std::f64::consts::PI * self.radius
    }
}

fn main() {
    let c = Circle { radius: 5.0 };
    println!("area: {:.2}", c.area());
}
```

## `self`, `&self`, `&mut self`

The first parameter determines how the method accesses the struct:

| Parameter | Meaning | Consumes the struct |
|-----------|---------|---------------------|
| `self` | Takes ownership | Yes |
| `&self` | Reads the struct | No |
| `&mut self` | Modifies the struct | No |

```rust
impl Rectangle {
    fn area(&self) -> f64 {           // read-only
        self.width * self.height
    }

    fn scale(&mut self, factor: f64) { // modifies
        self.width  *= factor;
        self.height *= factor;
    }

    fn to_string(self) -> String { // consumes
        format!("{}x{}", self.width, self.height)
    }
}
```

## Associated functions (constructors)

Functions that don't take `self` are called *associated functions*. They're used for constructors and utilities:

```rust
impl Rectangle {
    // Constructor: called with Rectangle::new(...)
    fn new(width: f64, height: f64) -> Rectangle {
        Rectangle { width, height }
    }

    fn square(side: f64) -> Rectangle {
        Rectangle { width: side, height: side }
    }
}

fn main() {
    let r = Rectangle::new(10.0, 5.0);
    let q = Rectangle::square(4.0);
}
```

## Multiple `impl` blocks

You can have multiple `impl` blocks for the same struct — they're equivalent to one:

```rust
impl Rectangle {
    fn area(&self) -> f64 { self.width * self.height }
}

impl Rectangle {
    fn can_contain(&self, other: &Rectangle) -> bool {
        self.width >= other.width && self.height >= other.height
    }
}
```

## Method chaining

If methods return `&mut Self`, you can chain them:

```rust
struct Builder {
    name:  String,
    value: i32,
}

impl Builder {
    fn new() -> Builder {
        Builder { name: String::new(), value: 0 }
    }

    fn name(mut self, n: &str) -> Self {
        self.name = n.to_string();
        self
    }

    fn value(mut self, v: i32) -> Self {
        self.value = v;
        self
    }
}

fn main() {
    let c = Builder::new()
        .name("test")
        .value(42);
    println!("{}: {}", c.name, c.value);
}
```
