# Constants and Shadowing

## Constants with `const`

Constants are values that **never change** and are evaluated at compile time. They are declared with `const` and must have an explicit type:

```rust
const MAX_POINTS: u32 = 100_000;
const PI: f64 = 3.14159265358979;
```

Differences from `let`:
- They can be declared in any scope, including global.
- They must have an explicit type.
- They can only be assigned to expressions evaluable at compile time.
- By convention: `SCREAMING_SNAKE_CASE`.

## Static variables with `static`

`static` is similar to `const` but has a fixed memory address for the entire execution:

```rust
static GREETING: &str = "Hello, world!";

fn main() {
    println!("{GREETING}");
}
```

Use `const` for most cases. Use `static` when you need a reference with `'static` lifetime.

## Shadowing

Rust allows declaring a new variable with the same name as a previous one. This is called *shadowing*:

```rust
fn main() {
    let x = 5;

    let x = x + 1;  // new variable x = 6

    {
        let x = x * 2;  // new x in this scope = 12
        println!("x in inner scope: {x}"); // 12
    }

    println!("x in outer scope: {x}"); // 6
}
```

### Shadowing vs `mut`

Shadowing is different from mutability. With shadowing you can **change the type**:

```rust
fn main() {
    let spaces = "   ";          // &str
    let spaces = spaces.len();   // usize — different type!

    println!("Spaces: {spaces}");
}
```

With `mut` this doesn't work:
```rust
fn main() {
    let mut spaces = "   ";
    // spaces = spaces.len(); // ERROR: incompatible types
}
```

### When to use shadowing?

- When you want to transform a value and don't need the original.
- To avoid names like `value_str`, `value_num` — just redeclare with the same name.
- To make step-by-step transformations within a block.
