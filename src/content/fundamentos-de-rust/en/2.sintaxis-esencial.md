# Essential Syntax

In this chapter we'll explore the fundamentals of Rust syntax: how to declare variables, the available data types, and the features that make Rust different from the start.

One of the first things you'll notice is that **Rust forces you to be explicit** — if something can change, you say so. If something can be an error, you handle it. The compiler is strict, but in return it guarantees that your program won't have an entire class of bugs in production.

This chapter covers:

- Variables and mutability (immutable by default)
- Primitive types (`i32`, `f64`, `bool`, `char`)
- Type inference
- Constants (`const` and `static`)
- Shadowing
