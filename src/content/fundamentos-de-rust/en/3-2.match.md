# match

`match` is Rust's *pattern matching* tool. It's like a `switch` on steroids: it's **exhaustive** (you must cover all cases) and can extract values from complex structures.

## Basic match

```rust
fn main() {
    let value = 4;

    match value {
        1 => println!("One"),
        2 => println!("Two"),
        3 => println!("Three"),
        _ => println!("Other"),  // _ is the wildcard (catch-all)
    }
}
```

If you forget the wildcard `_` and don't cover all possible values, the compiler fails:

```
error[E0004]: non-exhaustive patterns
```

## Multiple patterns and ranges

```rust
fn main() {
    let c = 'b';

    match c {
        'a' | 'e' | 'i' | 'o' | 'u' => println!("vowel"),
        'a'..='z' => println!("lowercase consonant"),
        'A'..='Z' => println!("uppercase consonant"),
        _ => println!("other character"),
    }
}
```

## match as an expression

Just like `if`, `match` returns a value:

```rust
fn main() {
    let number = 7;

    let kind = match number {
        n if n < 0 => "negative",
        0 => "zero",
        n if n % 2 == 0 => "positive even",
        _ => "positive odd",
    };

    println!("{number} is {kind}");
}
```

Extra conditions (`if n < 0`) are called **guards**.

## Destructuring in match

`match` can decompose tuples, structs, and enums:

```rust
fn main() {
    let point = (3, -2);

    match point {
        (0, 0) => println!("Origin"),
        (x, 0) => println!("On the X axis: {x}"),
        (0, y) => println!("On the Y axis: {y}"),
        (x, y) => println!("Point ({x}, {y})"),
    }
}
```

## Capturing the value with `@`

```rust
fn main() {
    let number = 5;

    match number {
        n @ 1..=9 => println!("Digit: {n}"),
        n @ 10..=99 => println!("Two digits: {n}"),
        _ => println!("Too large"),
    }
}
```

> **Key:** `match` is exhaustive by design. The compiler guarantees you never forgot a case.
