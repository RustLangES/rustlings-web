# Modules and Code Organization

As your code grows, you need to organize it. Rust uses a **module** system (`mod`) to create namespaces and control code visibility.

## The module system

### `mod` — defining a module

```rust
mod math {
    pub fn add(a: i32, b: i32) -> i32 {
        a + b
    }

    pub fn subtract(a: i32, b: i32) -> i32 {
        a - b
    }

    // Private by default — only accessible within the module
    fn helper() -> i32 {
        42
    }
}

fn main() {
    let result = math::add(5, 3);
    println!("{result}");
}
```

### Visibility with `pub`

By default, everything in Rust is **private**. Use `pub` to make it accessible:

```rust
mod geometry {
    pub struct Rectangle {
        pub width: f64,  // public field
        height: f64,     // private field
    }

    impl Rectangle {
        pub fn new(width: f64, height: f64) -> Rectangle {
            Rectangle { width, height }
        }

        pub fn area(&self) -> f64 {
            self.width * self.height  // can access `height` because we're in the same module
        }
    }
}

fn main() {
    let r = geometry::Rectangle::new(10.0, 5.0);
    println!("area: {}", r.area());
    println!("width: {}", r.width); // OK: it's pub
    // println!("{}", r.height);    // ERROR: it's private
}
```

## `use` — importing paths

To avoid writing the full path each time:

```rust
use std::collections::HashMap;
use std::collections::HashSet;

// Or multiple at once:
use std::collections::{HashMap, HashSet, BTreeMap};

// All public elements:
use std::collections::*;

// With alias:
use std::collections::HashMap as Map;

fn main() {
    let mut m: HashMap<&str, i32> = HashMap::new();
    m.insert("one", 1);
}
```

## Modules in separate files

For larger projects, each module can live in its own file.

### File structure

```
src/
├── main.rs
├── math.rs         ← math module
└── geometry/
    ├── mod.rs      ← geometry module
    ├── circle.rs   ← geometry::circle submodule
    └── rectangle.rs ← geometry::rectangle submodule
```

### `main.rs`

```rust
mod math;      // Rust will look for src/math.rs
mod geometry;  // Rust will look for src/geometry/mod.rs

use math::add;

fn main() {
    println!("{}", add(2, 3));
}
```

### `src/math.rs`

```rust
pub fn add(a: i32, b: i32) -> i32 {
    a + b
}
```

### `src/geometry/mod.rs`

```rust
pub mod circle;
pub mod rectangle;
```

## Absolute and relative paths

```rust
mod a {
    pub mod b {
        pub fn function() {
            println!("in a::b::function");
        }
    }
}

fn main() {
    // Absolute path (from crate root)
    crate::a::b::function();

    // With use
    use crate::a::b::function;
    function();
}
```

Within a module, you can use `super::` to go up one level:

```rust
mod parent {
    fn parent_function() {}

    mod child {
        fn child_function() {
            super::parent_function(); // accesses the parent
        }
    }
}
```

## Re-exporting with `pub use`

To simplify the public API of your library:

```rust
mod internal {
    pub struct Type { /* ... */ }
}

// Re-exports Type at the root level
pub use internal::Type;

// Users can do: use my_crate::Type;
// instead of: use my_crate::internal::Type;
```

## Recommended clean structure

```
my_project/
├── Cargo.toml
└── src/
    ├── main.rs         ← entry point, orchestrates modules
    ├── lib.rs          ← library public API (optional)
    ├── models.rs       ← data structs and enums
    ├── operations.rs   ← business logic
    └── utils.rs        ← common utilities
```

> **Golden rule:** keep `main.rs` small — let it only initialize the application and call well-organized modules. The logic lives in separate modules.
