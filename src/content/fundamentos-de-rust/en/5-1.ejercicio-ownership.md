# Exercise: Ownership and Move

The goal is to practice how Rust transfers ownership of values.

Each function **takes ownership** of its arguments — after calling them, the original value is consumed (moved). To keep access to the original data, you have two options: return the ownership or use `.clone()`.

## Functions to implement

```rust
// Takes ownership of the String and returns its uppercase version
fn en_mayusculas(texto: String) -> String

// Takes ownership of both strings and returns their concatenation
fn concatenar(a: String, b: String) -> String

// Returns the length and gives back ownership of the String
fn longitud(texto: String) -> (usize, String)

// Returns the original and an uppercase copy — without consuming the original
fn clonar_upper(texto: String) -> (String, String)
```

## Examples

```
en_mayusculas("hola rust")  → "HOLA RUST"

concatenar("hola", " mundo") → "hola mundo"

longitud("rustlings") → (9, "rustlings")

clonar_upper("hola") → ("hola", "HOLA")
```

## Hints

- `en_mayusculas`: use the `.to_uppercase()` method of `String`.
- `concatenar`: use `format!("{}{}", a, b)` to join without worrying about borrowing.
- `longitud`: first get `texto.len()` (doesn't consume), then return the pair `(len, texto)`.
- `clonar_upper`: use `.clone()` to get a copy, then call `.to_uppercase()` on the copy.
