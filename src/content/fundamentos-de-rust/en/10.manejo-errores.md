# Basic Error Handling

Rust has no exceptions. Instead, it uses `Result<T, E>` for recoverable errors and `panic!` for unrecoverable errors. This distinction is explicit and deliberate.

This chapter covers:

- `Result<T, E>` in depth
- `match` with `Result`
- `unwrap` and `expect` — when to use them and when not to
- The `?` operator for propagating errors
- Custom errors
