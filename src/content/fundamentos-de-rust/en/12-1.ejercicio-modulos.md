# Exercise: Modules and Visibility

Implement the content of the `conversion` module using `pub` to control what is visible from outside.

## What to implement

Inside the `conversion` module:

```rust
pub struct Temperatura {
    pub celsius: f64,
}

impl Temperatura {
    pub fn nueva(celsius: f64) -> Temperatura
    pub fn a_fahrenheit(&self) -> f64   // C × 9/5 + 32
    pub fn a_kelvin(&self) -> f64       // C + 273.15
    pub fn es_congelamiento(&self) -> bool  // true if celsius <= 0.0
}

pub fn celsius_a_fahrenheit(c: f64) -> f64
```

## Concepts practiced

- **`mod`** — encapsulates code in a module.
- **`pub`** — controls what is accessible from outside the module (struct, fields, methods, functions).
- **`use`** — imports `Temperatura` into the current scope so it can be used without the `conversion::` prefix.
- **Access with prefix** — `conversion::celsius_a_fahrenheit(...)` accesses a public function without `use`.

## Hints

- The `use conversion::Temperatura` is already written — this allows using `Temperatura` directly in `main` and in the tests.
- Without `pub`, fields and methods are not accessible from outside the module.
- Formulas: Fahrenheit = `celsius * 9.0 / 5.0 + 32.0`, Kelvin = `celsius + 273.15`.
