# Enums with Data

## Simple enum

```rust
#[derive(Debug)]
enum DayOfWeek {
    Monday,
    Tuesday,
    Wednesday,
    Thursday,
    Friday,
    Saturday,
    Sunday,
}

fn is_weekend(day: &DayOfWeek) -> bool {
    match day {
        DayOfWeek::Saturday | DayOfWeek::Sunday => true,
        _ => false,
    }
}
```

## Enums with data

Each variant can contain data, even of different types:

```rust
#[derive(Debug)]
enum Message {
    Quit,                           // no data
    Move { x: i32, y: i32 },       // anonymous struct
    Write(String),                  // tuple with one element
    ChangeColor(u8, u8, u8),       // tuple with three elements
}
```

This is much more expressive than using a struct with optional fields. Each variant is its own data type.

## Exhaustive `match` over enums

When you `match` over an enum, you must cover **all** variants:

```rust
fn process(msg: Message) {
    match msg {
        Message::Quit => {
            println!("Quitting...");
        }
        Message::Move { x, y } => {
            println!("Moving to ({x}, {y})");
        }
        Message::Write(text) => {
            println!("Text: {text}");
        }
        Message::ChangeColor(r, g, b) => {
            println!("Color: rgb({r}, {g}, {b})");
        }
    }
}
```

If you add a variant to the enum and forget to handle it in some `match`, the compiler tells you.

## Methods on enums

Enums can also have `impl` blocks:

```rust
impl Message {
    fn is_terminal(&self) -> bool {
        match self {
            Message::Quit => true,
            _ => false,
        }
    }
}
```

## `if let` — for a single pattern

When you're only interested in one variant:

```rust
fn main() {
    let msg = Message::Write(String::from("hello"));

    // Instead of match with _ => ()
    if let Message::Write(text) = msg {
        println!("Message received: {text}");
    }
}
```

## Enums as state machines

Enums are perfect for modeling states:

```rust
#[derive(Debug)]
enum ConnectionState {
    Disconnected,
    Connecting { host: String, port: u16 },
    Connected { id: u32 },
    Error(String),
}
```

> **Key:** enums with data let you model exactly the possible states of your system, without invalid values.
