# Loops: loop, while, and for

Rust has three types of loops, each with a different purpose.

## `loop` — Infinite loop

`loop` creates a loop that only ends with `break`. It's useful when you don't know in advance how many times to iterate:

```rust
fn main() {
    let mut counter = 0;

    loop {
        counter += 1;
        if counter == 3 { continue; }  // skip this iteration
        println!("counter: {counter}");
        if counter == 5 { break; }
    }
}
```

### `loop` as an expression — `break` with a value

```rust
fn main() {
    let mut attempts = 0;

    let result = loop {
        attempts += 1;
        if attempts == 10 {
            break attempts * 2;  // returns the value
        }
    };

    println!("Result: {result}"); // 20
}
```

## `while` — Conditional loop

```rust
fn main() {
    let mut number = 3;

    while number != 0 {
        println!("{number}!");
        number -= 1;
    }
    println!("LIFTOFF!");
}
```

## `for` — Iteration over collections

`for` is the most idiomatic loop in Rust:

```rust
fn main() {
    let array = [10, 20, 30, 40, 50];

    for element in array {
        println!("{element}");
    }
}
```

### Ranges with `for`

```rust
fn main() {
    for i in 1..4 {    // 1, 2, 3 (excludes 4)
        println!("{i}");
    }

    for i in 1..=4 {   // 1, 2, 3, 4 (includes 4)
        println!("{i}");
    }

    // Reverse
    for i in (1..=4).rev() {
        println!("{i}"); // 4, 3, 2, 1
    }
}
```

### Iterating with indices

```rust
fn main() {
    let fruits = ["apple", "banana", "cherry"];

    for (index, fruit) in fruits.iter().enumerate() {
        println!("{index}: {fruit}");
    }
}
```

## Labels for nested loops

If you have nested loops and need to `break` the outer one:

```rust
fn main() {
    'outer: for i in 0..5 {
        for j in 0..5 {
            if i == 2 && j == 2 {
                break 'outer; // exits the outer loop
            }
            println!("({i}, {j})");
        }
    }
}
```

> **Practical rule:**
> - Use `for` when iterating over a known collection or range.
> - Use `while` when the exit condition depends on state.
> - Use `loop` when you need to return a value from the loop or the exit condition is complex.
