# Option and Result

These two standard library enums are fundamental in Rust. They replace null values (`null`) and exceptions from other languages.

## `Option<T>` — values that may not exist

```rust
enum Option<T> {
    Some(T),   // there is a value
    None,      // there is no value
}
```

Rust has no `null`. Instead, when a value may or may not exist, you use `Option`:

```rust
fn find_user(id: u32) -> Option<String> {
    if id == 1 {
        Some(String::from("Alice"))
    } else {
        None
    }
}

fn main() {
    match find_user(1) {
        Some(name) => println!("Found: {name}"),
        None => println!("User not found"),
    }
}
```

### Useful `Option` methods

```rust
fn main() {
    let x: Option<i32> = Some(5);
    let y: Option<i32> = None;

    // unwrap: extracts the value or panics
    println!("{}", x.unwrap());         // 5

    // unwrap_or: default value
    println!("{}", y.unwrap_or(0));     // 0

    // map: transforms the value if it exists
    let double = x.map(|v| v * 2);
    println!("{:?}", double);           // Some(10)

    // is_some / is_none
    println!("{} {}", x.is_some(), y.is_none()); // true true
}
```

### `if let` with Option

```rust
fn main() {
    let number: Option<i32> = Some(7);

    if let Some(n) = number {
        println!("The number is: {n}");
    }

    // while let — process until there are no more
    let mut stack = vec![1, 2, 3];
    while let Some(top) = stack.pop() {
        println!("Processing: {top}");
    }
}
```

## `Result<T, E>` — operations that can fail

```rust
enum Result<T, E> {
    Ok(T),   // success with value T
    Err(E),  // error with value E
}
```

```rust
fn parse_number(s: &str) -> Result<i32, String> {
    s.parse::<i32>()
     .map_err(|_| format!("'{s}' is not a valid number"))
}

fn main() {
    match parse_number("42") {
        Ok(n)  => println!("Number: {n}"),
        Err(e) => println!("Error: {e}"),
    }

    match parse_number("abc") {
        Ok(n)  => println!("Number: {n}"),
        Err(e) => println!("Error: {e}"),
    }
}
```

### Useful `Result` methods

```rust
fn main() {
    let ok: Result<i32, &str> = Ok(10);
    let err: Result<i32, &str> = Err("failed");

    // unwrap / expect
    println!("{}", ok.unwrap());
    // err.unwrap();  // panic!

    println!("{}", err.unwrap_or(-1));  // -1

    // map — transforms the Ok
    let double = ok.map(|v| v * 2);
    println!("{:?}", double); // Ok(20)

    // is_ok / is_err
    println!("{} {}", ok.is_ok(), err.is_err()); // true true
}
```

### The `?` operator (introduction)

The `?` operator propagates errors automatically:

```rust
fn read_and_parse(text: &str) -> Result<i32, String> {
    let number: i32 = text.trim()
        .parse()
        .map_err(|_| format!("not a number: '{text}'"))?;
    Ok(number * 2)
}
```

If `parse()` fails, `?` returns the `Err` immediately. If it succeeds, it continues with the value.

> **Summary:** `Option` for optional values, `Result` for operations that can fail. Both force you to explicitly handle cases — no runtime surprises.
