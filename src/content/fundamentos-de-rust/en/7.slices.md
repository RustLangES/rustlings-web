# Slices: References to Portions

A slice is a **reference to a contiguous portion** of a collection — without copying the data. Slices have two parts: a pointer to the first element and a length.

## String slices (`&str`)

A `&str` is a string slice — a view of a portion of text:

```rust
fn main() {
    let s = String::from("hello world");

    let hello = &s[0..5];   // "hello"
    let world = &s[6..11];  // "world"
    let all   = &s[..];     // "hello world"

    println!("{hello} | {world} | {all}");
}
```

Text literals like `"hello"` are directly `&str` — slices pointing to data in the program's binary.

## Why `&str` is better than `&String` in parameters

```rust
// Only accepts &String
fn length_v1(s: &String) -> usize { s.len() }

// Accepts &String, &str, literals — more flexible
fn length_v2(s: &str) -> usize { s.len() }

fn main() {
    let s = String::from("hello");

    length_v1(&s);        // OK
    // length_v1("hello"); // ERROR: wrong type

    length_v2(&s);        // OK (&String coerces to &str)
    length_v2("hello");   // OK (literal is already &str)
}
```

## Array slices (`&[T]`)

Slices work the same way with arrays and vectors:

```rust
fn main() {
    let a = [1, 2, 3, 4, 5];

    let slice = &a[1..3]; // [2, 3]
    println!("{:?}", slice);
    println!("length: {}", slice.len());

    // Generic function over slices
    print_slice(&a);
    print_slice(&a[2..]);
}

fn print_slice(s: &[i32]) {
    for x in s {
        print!("{x} ");
    }
    println!();
}
```

## Slices and ownership

Slices are references — they don't own the data:

```rust
fn main() {
    let v = vec![10, 20, 30, 40, 50];

    let slice = &v[1..4]; // immutable reference to v

    // v.push(60); // ERROR: can't modify v while slice exists
    println!("{:?}", slice);
    // Here the slice usage ends, now you could modify v
}
```

## Useful slice operations

```rust
fn main() {
    let numbers = [3, 1, 4, 1, 5, 9, 2, 6];
    let slice = &numbers[..];

    println!("length: {}", slice.len());
    println!("first: {:?}", slice.first());
    println!("last: {:?}", slice.last());
    println!("contains 5: {}", slice.contains(&5));

    // Iteration
    let sum: i32 = slice.iter().sum();
    let max = slice.iter().max().unwrap();
    println!("sum={sum}, max={max}");
}
```

## Implicit lifetime rules (lifetime elision)

When a function takes a slice and returns one, Rust infers that the returned slice points to the parameter's data:

```rust
// The lifetime is implicit: the returned & points to `s`'s data
fn first_word(s: &str) -> &str {
    match s.find(' ') {
        Some(i) => &s[..i],
        None => s,
    }
}

fn main() {
    let text = String::from("hello world");
    let word = first_word(&text);
    println!("{word}"); // "hello"
    // text can't be modified here because `word` is referencing it
}
```

> **Summary:** `&[T]` and `&str` are the slices of arrays and strings respectively. Use them in functions instead of references to full collections — they're more flexible and more idiomatic in Rust.
