# Exercise: Enums in Action

Two exercises that use enums with exhaustive `match` and data-carrying variants.

## Part 1: Coins

```rust
enum Moneda { Centavo, Cinco, Diez, Veinticinco }

// Returns the value in cents of each coin
fn valor_centavos(moneda: &Moneda) -> u32

// Sums the total value of a slice of coins
fn total_centavos(monedas: &[Moneda]) -> u32
```

```
valor_centavos(&Moneda::Veinticinco) → 25
total_centavos(&[Veinticinco, Diez, Cinco, Centavo, Centavo]) → 42
```

## Part 2: Temperature

The `Temperatura` enum can represent degrees in three different scales:

```rust
enum Temperatura {
    Celsius(f64),
    Fahrenheit(f64),
    Kelvin(f64),
}

// Converts any temperature to Celsius
fn a_celsius(temp: &Temperatura) -> f64

// Returns true if the temperature equals 0°C or below
fn es_congelamiento(temp: &Temperatura) -> bool
```

Conversion formulas:
```
Fahrenheit → Celsius:  (F - 32) × 5/9
Kelvin → Celsius:       K - 273.15
```

## Hints

- `valor_centavos`: a simple `match` over the variants.
- `total_centavos`: iterate the slice with `for` and accumulate with `valor_centavos`.
- `a_celsius`: use `match temp` with patterns `Temperatura::Celsius(c) => *c`, etc.
- `es_congelamiento`: call `a_celsius(temp)` and compare with `0.0`.
