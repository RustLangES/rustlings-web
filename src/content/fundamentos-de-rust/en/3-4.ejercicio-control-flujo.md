# Exercise: Control Flow

Apply `if`, `match`, and loops to solve these problems using only primitive types.

## Functions to implement

```rust
// Returns -1 if n is negative, 0 if zero, 1 if positive
fn clasificar(n: i32) -> i32

// Sums all even numbers from 1 to `limite` (inclusive)
fn suma_pares_hasta(limite: u32) -> u32

// Counts how many numbers from 1 to `limite` (inclusive) are divisible by `factor`
fn contar_multiplos(limite: u32, factor: u32) -> u32

// Calculates base^exp using a loop (without using pow)
fn potencia(base: i64, exp: u32) -> i64
```

## Examples

```
clasificar(-5) → -1
clasificar(0)  → 0
clasificar(3)  → 1

suma_pares_hasta(10) → 30   (2+4+6+8+10)
suma_pares_hasta(1)  → 0

contar_multiplos(10, 2) → 5  (2,4,6,8,10)
contar_multiplos(10, 3) → 3  (3,6,9)

potencia(2, 10) → 1024
potencia(5, 0)  → 1
```

## Hints

- `clasificar`: use `match n` with guards (`n if n < 0 => -1`) or an `if/else if/else`.
- `suma_pares_hasta`: use `for n in 1..=limite` and filter with `if n % 2 == 0`.
- `contar_multiplos`: similar, increment a counter when `n % factor == 0`.
- `potencia`: start with `result = 1` and multiply in a loop `exp` times.
