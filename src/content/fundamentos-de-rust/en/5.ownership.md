# Ownership: The Heart of Rust

*Ownership* is the mechanism Rust uses to manage memory **without a garbage collector and without memory errors**. It is the most distinctive feature of the language.

## The three rules of Ownership

1. **Each value has exactly one owner.**
2. **There can only be one owner at a time.**
3. **When the owner goes out of scope, the value is dropped.**

## Stack vs Heap

This distinction is fundamental to understanding ownership:

- **Stack:** size known at compile time, very fast access. Types: `i32`, `bool`, `char`, tuples of Copy types, etc.
- **Heap:** dynamic size, slower access. Types: `String`, `Vec`, `Box`, etc.

```rust
fn main() {
    let x = 5;            // lives on the stack
    let s = String::from("hello"); // data on the heap, pointer on the stack
} // Rust calls drop() automatically here and frees the memory
```

## Copy types vs Move types

Simple stack types implement the `Copy` trait — they are duplicated automatically:

```rust
fn main() {
    let a = 42;
    let b = a;   // COPY: a and b are independent
    println!("{a} {b}"); // both work
}
```

Heap types do a **move** (ownership transfer):

```rust
fn main() {
    let s1 = String::from("hello");
    let s2 = s1;   // MOVE: s1 no longer exists

    // println!("{s1}"); // ERROR: borrow of moved value: `s1`
    println!("{s2}");
}
```

Why? Rust prevents two variables from pointing to the same data on the heap — that would cause a *double free* when going out of scope.

## Clone — explicit heap copy

If you need an independent copy:

```rust
fn main() {
    let s1 = String::from("hello");
    let s2 = s1.clone(); // deep copy of the heap
    println!("{s1} {s2}"); // both valid
}
```

`clone()` is costly — only use it when you need it.

## Ownership and functions

Passing a value to a function **moves** or **copies** just like assignment:

```rust
fn take_ownership(s: String) {
    println!("{s}");
} // s goes out of scope and is dropped

fn make_copy(x: i32) {
    println!("{x}");
} // x goes out of scope, nothing special happens

fn main() {
    let s = String::from("world");
    take_ownership(s);
    // println!("{s}"); // ERROR: s was moved

    let x = 5;
    make_copy(x);
    println!("{x}"); // OK: i32 implements Copy
}
```

## Returning values transfers ownership

```rust
fn create_string() -> String {
    String::from("new") // ownership is transferred to the caller
}

fn main() {
    let s = create_string(); // s is now the owner
    println!("{s}");
} // s goes out of scope, is dropped
```

## Scope and drop

```rust
fn main() {
    {
        let s = String::from("hello"); // s enters scope
        println!("{s}");
    } // s goes out of scope → Rust calls drop() automatically
      // heap memory is freed here

    // println!("{s}"); // ERROR: s is not in scope
}
```

This pattern is called **RAII** (Resource Acquisition Is Initialization) — resources are freed when the owner goes out of scope.

> **Summary:** Rust guarantees that each value is freed exactly once, automatically, without a garbage collector. This eliminates an entire class of memory bugs.
