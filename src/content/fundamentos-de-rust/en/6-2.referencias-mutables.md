# Mutable References and the Borrow Checker Rules

## Mutable references

To modify a value through a reference, you need `&mut`:

```rust
fn change(s: &mut String) {
    s.push_str(", world!");
}

fn main() {
    let mut s = String::from("hello");
    change(&mut s);
    println!("{s}"); // "hello, world!"
}
```

The original variable must also be `mut`.

## The borrow checker rules

The borrow checker is the part of the compiler that enforces borrowing rules:

### Rule 1: Only one mutable reference at a time

```rust
fn main() {
    let mut s = String::from("hello");

    let r1 = &mut s;
    // let r2 = &mut s; // ERROR: cannot borrow `s` as mutable more than once

    println!("{r1}");
    // After r1 is no longer used, we can create another:
    let r2 = &mut s;
    println!("{r2}");
}
```

### Rule 2: You can't mix immutable and mutable references simultaneously

```rust
fn main() {
    let mut s = String::from("hello");

    let r1 = &s;     // OK
    let r2 = &s;     // OK
    // let r3 = &mut s; // ERROR while r1 and r2 are active

    println!("{r1} {r2}");
    // r1 and r2 are no longer used — now we can make it mutable:
    let r3 = &mut s;
    println!("{r3}");
}
```

### Why these rules?

They prevent **data races** — concurrent accesses that can corrupt data:
- Multiple simultaneous reads: ✅ safe
- One write without simultaneous reads: ✅ safe
- Simultaneous write + read: ❌ data race

## Common errors and their solutions

### Error: dangling reference

```rust
// ERROR: the value goes out of scope before the reference
// fn create_reference() -> &String {
//     let s = String::from("hello");
//     &s  // ERROR: s is destroyed when leaving the function
// }

// Solution: return the value, not a reference
fn create_string() -> String {
    String::from("hello")
}
```

### Error: multiple mutable references

```rust
fn main() {
    let mut v = vec![1, 2, 3];

    // This fails:
    // let first = &v[0];
    // v.push(4);           // ERROR: v might be reallocated in memory
    // println!("{first}");

    // Solution: don't keep references while modifying
    v.push(4);
    let first = &v[0];   // OK: take reference after modifying
    println!("{first}");
}
```

## Summary of rules

| Reference type | Simultaneous count | Allows modification |
|----------------|-------------------|---------------------|
| `&T` (immutable) | Any | No |
| `&mut T` (mutable) | Exactly one | Yes |

> **Key:** the borrow checker is not your enemy — it's protecting you from errors that would be difficult to debug at runtime.
