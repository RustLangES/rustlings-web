# Variables and Mutability

In Rust, **all variables are immutable by default**. This is one of the most important design decisions of the language — it forces you to be explicit when you want something to change.

## Immutable variables

```rust
fn main() {
    let x = 5;
    // x = 6; // COMPILATION ERROR!
    println!("The value of x is: {x}");
}
```

If you try to reassign `x`, the compiler stops you cold:

```
error[E0384]: cannot assign twice to immutable variable `x`
```

## Mutable variables with `mut`

To make a variable modifiable, use `mut`:

```rust
fn main() {
    let mut x = 5;
    println!("Before:  {x}");
    x = 6;
    println!("After: {x}");
}
```

## Why immutable by default?

1. **Safety** — the compiler detects accidental modifications.
2. **Concurrency** — immutable data can be shared between threads without risk.
3. **Clarity** — when you see `let mut`, you know that value is going to change.

## Type inference

Rust can infer the type in most cases — it's not always necessary to annotate it:

```rust
fn main() {
    let x = 42;          // Rust infers i32
    let pi = 3.14;       // Rust infers f64
    let active = true;   // Rust infers bool
    let letter = 'R';    // Rust infers char

    // You can also be explicit:
    let y: i64 = 100;
}
```

> **Golden rule:** Always start with `let`. Add `mut` only when the logic requires it.
