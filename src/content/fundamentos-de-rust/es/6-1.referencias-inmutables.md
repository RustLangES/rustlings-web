# Referencias Inmutables

Una referencia (`&T`) permite acceder a un valor sin tomar su ownership. El `&` crea una referencia; este proceso se llama *borrowing*.

## Borrowing básico

```rust
fn calcular_longitud(s: &String) -> usize {
    s.len()
} // s es una referencia, no el dueño — no se libera nada

fn main() {
    let s1 = String::from("hola");

    let len = calcular_longitud(&s1); // prestamos s1

    println!("La longitud de '{s1}' es {len}"); // s1 sigue siendo válido
}
```

## Múltiples referencias inmutables

Puedes tener **tantas referencias inmutables como quieras** al mismo tiempo:

```rust
fn main() {
    let s = String::from("hola");

    let r1 = &s;
    let r2 = &s;
    let r3 = &s;

    println!("{r1} {r2} {r3}"); // todos válidos simultáneamente
}
```

## Las referencias no modifican el valor

Las referencias inmutables son solo de lectura:

```rust
fn intentar_modificar(s: &String) {
    // s.push_str("!"); // ERROR: cannot borrow `*s` as mutable
    println!("{s}"); // solo lectura: OK
}
```

## Por qué usar referencias en lugar de mover

```rust
// Sin referencias: necesitas retornar el String para seguir usándolo
fn longitud_sin_ref(s: String) -> (String, usize) {
    let len = s.len();
    (s, len) // devolvemos s para no perderlo
}

// Con referencias: mucho más limpio
fn longitud(s: &String) -> usize {
    s.len()
}

fn main() {
    let s = String::from("hola");
    let len = longitud(&s); // s sigue siendo nuestro
    println!("'{s}' tiene {len} caracteres");
}
```

## `&str` vs `&String`

Para parámetros de funciones, prefiere `&str` sobre `&String` — acepta tanto `String` como literales:

```rust
fn saludar(nombre: &str) {
    println!("¡Hola, {nombre}!");
}

fn main() {
    let s = String::from("Alice");
    saludar(&s);           // &String se convierte a &str
    saludar("Bob");        // &str directo
}
```

> **Regla:** usa `&str` en parámetros de funciones cuando solo necesitas leer texto. Es más flexible que `&String`.
