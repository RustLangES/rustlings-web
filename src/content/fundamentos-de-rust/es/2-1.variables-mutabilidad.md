# Variables y Mutabilidad

En Rust, **todas las variables son inmutables por defecto**. Esta es una de las decisiones de diseño más importantes del lenguaje — te obliga a ser explícito cuando quieres que algo cambie.

## Variables inmutables

```rust
fn main() {
    let x = 5;
    // x = 6; // ¡ERROR de compilación!
    println!("El valor de x es: {x}");
}
```

Si intentas reasignar `x`, el compilador te para en seco:

```
error[E0384]: cannot assign twice to immutable variable `x`
```

## Variables mutables con `mut`

Para que una variable sea modificable, usa `mut`:

```rust
fn main() {
    let mut x = 5;
    println!("Antes:  {x}");
    x = 6;
    println!("Después: {x}");
}
```

## ¿Por qué inmutable por defecto?

1. **Seguridad** — el compilador detecta modificaciones accidentales.
2. **Concurrencia** — los datos inmutables se pueden compartir entre hilos sin riesgo.
3. **Claridad** — cuando ves `let mut`, sabes que ese valor va a cambiar.

## Inferencia de tipos

Rust puede inferir el tipo en la mayoría de casos — no siempre hace falta anotarlo:

```rust
fn main() {
    let x = 42;          // Rust infiere i32
    let pi = 3.14;       // Rust infiere f64
    let activo = true;   // Rust infiere bool
    let letra = 'R';     // Rust infiere char

    // También puedes ser explícito:
    let y: i64 = 100;
}
```

> **Regla de oro:** Empieza siempre con `let`. Añade `mut` solo cuando la lógica lo requiera.
