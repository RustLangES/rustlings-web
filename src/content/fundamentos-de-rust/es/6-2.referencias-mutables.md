# Referencias Mutables y las Reglas del Borrow Checker

## Referencias mutables

Para modificar un valor a través de una referencia, necesitas `&mut`:

```rust
fn cambiar(s: &mut String) {
    s.push_str(", mundo!");
}

fn main() {
    let mut s = String::from("hola");
    cambiar(&mut s);
    println!("{s}"); // "hola, mundo!"
}
```

La variable original también debe ser `mut`.

## Las reglas del borrow checker

El borrow checker es la parte del compilador que hace cumplir las reglas de borrowing:

### Regla 1: Una sola referencia mutable a la vez

```rust
fn main() {
    let mut s = String::from("hola");

    let r1 = &mut s;
    // let r2 = &mut s; // ERROR: cannot borrow `s` as mutable more than once

    println!("{r1}");
    // Después de que r1 deja de usarse, podemos crear otra:
    let r2 = &mut s;
    println!("{r2}");
}
```

### Regla 2: No puedes mezclar inmutables y mutables simultáneamente

```rust
fn main() {
    let mut s = String::from("hola");

    let r1 = &s;     // OK
    let r2 = &s;     // OK
    // let r3 = &mut s; // ERROR mientras r1 y r2 estén activas

    println!("{r1} {r2}");
    // r1 y r2 ya no se usan — ahora sí podemos hacer mutable:
    let r3 = &mut s;
    println!("{r3}");
}
```

### ¿Por qué estas reglas?

Previenen **data races** — accesos concurrentes que pueden corromper datos:
- Múltiples lecturas simultáneas: ✅ seguro
- Una escritura sin lecturas simultáneas: ✅ seguro
- Escritura + lectura simultáneas: ❌ data race

## Errores comunes y sus soluciones

### Error: dangling reference

```rust
// ERROR: el valor sale del scope antes que la referencia
// fn crear_referencia() -> &String {
//     let s = String::from("hola");
//     &s  // ERROR: s se destruye al salir de la función
// }

// Solución: retornar el valor, no una referencia
fn crear_string() -> String {
    String::from("hola")
}
```

### Error: múltiples referencias mutables

```rust
fn main() {
    let mut v = vec![1, 2, 3];

    // Esto falla:
    // let primero = &v[0];
    // v.push(4);           // ERROR: v se puede reubicar en memoria
    // println!("{primero}");

    // Solución: no mantener referencias mientras modificas
    v.push(4);
    let primero = &v[0];   // OK: tomamos referencia después de modificar
    println!("{primero}");
}
```

## Resumen de las reglas

| Tipo de referencia | Cantidad simultánea | Permite modificar |
|-------------------|---------------------|-------------------|
| `&T` (inmutable) | Cualquiera | No |
| `&mut T` (mutable) | Exactamente una | Sí |

> **Clave:** el borrow checker no es el enemigo — te está protegiendo de errores que serían difíciles de depurar en tiempo de ejecución.
