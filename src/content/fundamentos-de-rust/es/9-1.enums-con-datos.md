# Enums con Datos

## Enum simple

```rust
#[derive(Debug)]
enum DiaSemana {
    Lunes,
    Martes,
    Miercoles,
    Jueves,
    Viernes,
    Sabado,
    Domingo,
}

fn es_fin_de_semana(dia: &DiaSemana) -> bool {
    match dia {
        DiaSemana::Sabado | DiaSemana::Domingo => true,
        _ => false,
    }
}
```

## Enums con datos

Cada variante puede contener datos, incluso de tipos diferentes:

```rust
#[derive(Debug)]
enum Mensaje {
    Salir,                         // sin datos
    Mover { x: i32, y: i32 },     // struct anónimo
    Escribir(String),              // tupla con un elemento
    CambiarColor(u8, u8, u8),     // tupla con tres elementos
}
```

Esto es mucho más expresivo que usar una struct con campos opcionales. Cada variante es su propio tipo de dato.

## `match` exhaustivo sobre enums

Cuando haces `match` sobre un enum, debes cubrir **todas** las variantes:

```rust
fn procesar(msg: Mensaje) {
    match msg {
        Mensaje::Salir => {
            println!("Saliendo...");
        }
        Mensaje::Mover { x, y } => {
            println!("Moviéndose a ({x}, {y})");
        }
        Mensaje::Escribir(texto) => {
            println!("Texto: {texto}");
        }
        Mensaje::CambiarColor(r, g, b) => {
            println!("Color: rgb({r}, {g}, {b})");
        }
    }
}
```

Si añades una variante al enum y olvidas manejarla en algún `match`, el compilador te lo dice.

## Métodos en enums

Los enums también pueden tener bloques `impl`:

```rust
impl Mensaje {
    fn es_terminal(&self) -> bool {
        match self {
            Mensaje::Salir => true,
            _ => false,
        }
    }
}
```

## `if let` — para un solo patrón

Cuando solo te interesa una variante:

```rust
fn main() {
    let msg = Mensaje::Escribir(String::from("hola"));

    // En lugar de match con _ => ()
    if let Mensaje::Escribir(texto) = msg {
        println!("Mensaje recibido: {texto}");
    }
}
```

## Enums como máquinas de estado

Los enums son perfectos para modelar estados:

```rust
#[derive(Debug)]
enum EstadoConexion {
    Desconectado,
    Conectando { host: String, puerto: u16 },
    Conectado { id: u32 },
    Error(String),
}
```

> **Clave:** los enums con datos te permiten modelar exactamente los estados posibles de tu sistema, sin valores inválidos.
