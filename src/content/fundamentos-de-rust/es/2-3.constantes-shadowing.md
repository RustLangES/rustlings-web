# Constantes y Shadowing

## Constantes con `const`

Las constantes son valores que **nunca cambian** y se evalúan en tiempo de compilación. Se declaran con `const` y deben llevar tipo explícito:

```rust
const MAX_PUNTOS: u32 = 100_000;
const PI: f64 = 3.14159265358979;
```

Diferencias con `let`:
- Se pueden declarar en cualquier scope, incluido el global.
- Deben tener tipo explícito.
- Solo pueden asignarse a expresiones evaluables en tiempo de compilación.
- Por convención: `SCREAMING_SNAKE_CASE`.

## Variables estáticas con `static`

`static` es similar a `const` pero tiene una dirección de memoria fija durante toda la ejecución:

```rust
static SALUDO: &str = "Hola, mundo!";

fn main() {
    println!("{SALUDO}");
}
```

Usa `const` para la mayoría de casos. Usa `static` cuando necesitas una referencia con lifetime `'static`.

## Shadowing (Sombreado)

Rust permite declarar una nueva variable con el mismo nombre que una anterior. Esto se llama *shadowing*:

```rust
fn main() {
    let x = 5;

    let x = x + 1;  // nueva variable x = 6

    {
        let x = x * 2;  // nueva x en este scope = 12
        println!("x en scope interno: {x}"); // 12
    }

    println!("x en scope externo: {x}"); // 6
}
```

### Shadowing vs `mut`

El shadowing es diferente de la mutabilidad. Con shadowing puedes **cambiar el tipo**:

```rust
fn main() {
    let espacios = "   ";          // &str
    let espacios = espacios.len(); // usize — ¡tipo diferente!

    println!("Espacios: {espacios}");
}
```

Con `mut` esto no funciona:
```rust
fn main() {
    let mut espacios = "   ";
    // espacios = espacios.len(); // ERROR: tipos incompatibles
}
```

### ¿Cuándo usar shadowing?

- Cuando quieres transformar un valor y no necesitas el original.
- Para evitar nombres como `valor_str`, `valor_num` — simplemente redeclara con el mismo nombre.
- Para hacer transformaciones en pasos dentro de un bloque.
