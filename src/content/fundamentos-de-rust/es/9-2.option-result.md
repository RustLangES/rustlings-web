# Option y Result

Estos dos enums de la librería estándar son fundamentales en Rust. Reemplazan los valores nulos (`null`) y las excepciones de otros lenguajes.

## `Option<T>` — valores que pueden no existir

```rust
enum Option<T> {
    Some(T),   // hay un valor
    None,      // no hay valor
}
```

Rust no tiene `null`. En su lugar, cuando un valor puede o no existir, usas `Option`:

```rust
fn buscar_usuario(id: u32) -> Option<String> {
    if id == 1 {
        Some(String::from("Alice"))
    } else {
        None
    }
}

fn main() {
    match buscar_usuario(1) {
        Some(nombre) => println!("Encontrado: {nombre}"),
        None => println!("Usuario no encontrado"),
    }
}
```

### Métodos útiles de `Option`

```rust
fn main() {
    let x: Option<i32> = Some(5);
    let y: Option<i32> = None;

    // unwrap: extrae el valor o panic
    println!("{}", x.unwrap());         // 5

    // unwrap_or: valor por defecto
    println!("{}", y.unwrap_or(0));     // 0

    // map: transforma el valor si existe
    let doble = x.map(|v| v * 2);
    println!("{:?}", doble);            // Some(10)

    // is_some / is_none
    println!("{} {}", x.is_some(), y.is_none()); // true true
}
```

### `if let` con Option

```rust
fn main() {
    let numero: Option<i32> = Some(7);

    if let Some(n) = numero {
        println!("El número es: {n}");
    }

    // while let — procesar hasta que no haya más
    let mut pila = vec![1, 2, 3];
    while let Some(tope) = pila.pop() {
        println!("Procesando: {tope}");
    }
}
```

## `Result<T, E>` — operaciones que pueden fallar

```rust
enum Result<T, E> {
    Ok(T),   // éxito con valor T
    Err(E),  // error con valor E
}
```

```rust
fn parsear_numero(s: &str) -> Result<i32, String> {
    s.parse::<i32>()
     .map_err(|_| format!("'{s}' no es un número válido"))
}

fn main() {
    match parsear_numero("42") {
        Ok(n)  => println!("Número: {n}"),
        Err(e) => println!("Error: {e}"),
    }

    match parsear_numero("abc") {
        Ok(n)  => println!("Número: {n}"),
        Err(e) => println!("Error: {e}"),
    }
}
```

### Métodos útiles de `Result`

```rust
fn main() {
    let ok: Result<i32, &str> = Ok(10);
    let err: Result<i32, &str> = Err("falló");

    // unwrap / expect
    println!("{}", ok.unwrap());
    // err.unwrap();  // panic!

    println!("{}", err.unwrap_or(-1));  // -1

    // map — transforma el Ok
    let doble = ok.map(|v| v * 2);
    println!("{:?}", doble); // Ok(20)

    // is_ok / is_err
    println!("{} {}", ok.is_ok(), err.is_err()); // true true
}
```

### El operador `?` (introducción)

El operador `?` propaga errores automáticamente:

```rust
fn leer_y_parsear(texto: &str) -> Result<i32, String> {
    let numero: i32 = texto.trim()
        .parse()
        .map_err(|_| format!("no es un número: '{texto}'"))?;
    Ok(numero * 2)
}
```

Si el `parse()` falla, `?` retorna el `Err` inmediatamente. Si tiene éxito, continúa con el valor.

> **Resumen:** `Option` para valores opcionales, `Result` para operaciones que pueden fallar. Ambos te obligan a manejar explícitamente los casos — no hay sorpresas en tiempo de ejecución.
