# Ownership: El Corazón de Rust

El *ownership* (propiedad) es el mecanismo que Rust usa para gestionar la memoria **sin garbage collector y sin errores de memoria**. Es la característica más distintiva del lenguaje.

## Las tres reglas de Ownership

1. **Cada valor tiene exactamente un propietario.**
2. **Solo puede haber un propietario a la vez.**
3. **Cuando el propietario sale del scope, el valor se libera.**

## Stack vs Heap

Esta distinción es fundamental para entender ownership:

- **Stack:** tamaño conocido en compilación, acceso muy rápido. Tipos: `i32`, `bool`, `char`, tuplas de tipos Copy, etc.
- **Heap:** tamaño dinámico, acceso más lento. Tipos: `String`, `Vec`, `Box`, etc.

```rust
fn main() {
    let x = 5;           // vive en el stack
    let s = String::from("hola"); // datos en el heap, puntero en el stack
} // aquí Rust llama drop() automáticamente y libera la memoria
```

## Tipos Copy vs Move

Los tipos simples del stack implementan el trait `Copy` — se duplican automáticamente:

```rust
fn main() {
    let a = 42;
    let b = a;   // COPIA: a y b son independientes
    println!("{a} {b}"); // ambos funcionan
}
```

Los tipos del heap hacen **move** (transferencia de propiedad):

```rust
fn main() {
    let s1 = String::from("hola");
    let s2 = s1;   // MOVE: s1 ya no existe

    // println!("{s1}"); // ERROR: borrow of moved value: `s1`
    println!("{s2}");
}
```

¿Por qué? Rust evita que dos variables apunten al mismo dato en el heap — eso causaría un *double free* al salir del scope.

## Clone — copia explícita del heap

Si necesitas una copia independiente:

```rust
fn main() {
    let s1 = String::from("hola");
    let s2 = s1.clone(); // copia profunda del heap
    println!("{s1} {s2}"); // ambos válidos
}
```

`clone()` es costoso — solo úsalo cuando lo necesites.

## Ownership y funciones

Pasar un valor a una función **mueve** o **copia** igual que la asignación:

```rust
fn tomar_ownership(s: String) {
    println!("{s}");
} // s sale del scope y se libera

fn hacer_copia(x: i32) {
    println!("{x}");
} // x sale del scope, no pasa nada especial

fn main() {
    let s = String::from("mundo");
    tomar_ownership(s);
    // println!("{s}"); // ERROR: s fue movido

    let x = 5;
    hacer_copia(x);
    println!("{x}"); // OK: i32 implementa Copy
}
```

## Retornar valores transfiere ownership

```rust
fn crear_string() -> String {
    String::from("nuevo") // ownership se transfiere al caller
}

fn main() {
    let s = crear_string(); // s ahora es el propietario
    println!("{s}");
} // s sale de scope, se libera
```

## Scope y drop

```rust
fn main() {
    {
        let s = String::from("hola"); // s entra en scope
        println!("{s}");
    } // s sale del scope → Rust llama drop() automáticamente
      // la memoria del heap se libera aquí

    // println!("{s}"); // ERROR: s no está en scope
}
```

Este patrón se llama **RAII** (Resource Acquisition Is Initialization) — los recursos se liberan cuando el propietario sale del scope.

> **Resumen:** Rust garantiza que cada valor se libera exactamente una vez, automáticamente, sin garbage collector. Esto elimina toda una clase de bugs de memoria.
