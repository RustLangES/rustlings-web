# Módulos y Organización del Código

A medida que tu código crece, necesitas organizarlo. Rust usa un sistema de **módulos** (`mod`) para crear namespaces y controlar la visibilidad del código.

## El sistema de módulos

### `mod` — definir un módulo

```rust
mod matematicas {
    pub fn sumar(a: i32, b: i32) -> i32 {
        a + b
    }

    pub fn restar(a: i32, b: i32) -> i32 {
        a - b
    }

    // Privado por defecto — solo accesible dentro del módulo
    fn helper() -> i32 {
        42
    }
}

fn main() {
    let resultado = matematicas::sumar(5, 3);
    println!("{resultado}");
}
```

### Visibilidad con `pub`

Por defecto, todo en Rust es **privado**. Usa `pub` para hacerlo accesible:

```rust
mod geometria {
    pub struct Rectangulo {
        pub ancho: f64,  // campo público
        alto: f64,       // campo privado
    }

    impl Rectangulo {
        pub fn nuevo(ancho: f64, alto: f64) -> Rectangulo {
            Rectangulo { ancho, alto }
        }

        pub fn area(&self) -> f64 {
            self.ancho * self.alto  // puede acceder a `alto` porque está en el mismo módulo
        }
    }
}

fn main() {
    let r = geometria::Rectangulo::nuevo(10.0, 5.0);
    println!("área: {}", r.area());
    println!("ancho: {}", r.ancho); // OK: es pub
    // println!("{}", r.alto);      // ERROR: es privado
}
```

## `use` — importar rutas

Para evitar escribir la ruta completa cada vez:

```rust
use std::collections::HashMap;
use std::collections::HashSet;

// O varias a la vez:
use std::collections::{HashMap, HashSet, BTreeMap};

// Todos los elementos públicos:
use std::collections::*;

// Con alias:
use std::collections::HashMap as Mapa;

fn main() {
    let mut m: HashMap<&str, i32> = HashMap::new();
    m.insert("uno", 1);
}
```

## Módulos en archivos separados

Para proyectos más grandes, cada módulo puede vivir en su propio archivo.

### Estructura de archivos

```
src/
├── main.rs
├── matematicas.rs      ← módulo matematicas
└── geometria/
    ├── mod.rs          ← módulo geometria
    ├── circulo.rs      ← submódulo geometria::circulo
    └── rectangulo.rs   ← submódulo geometria::rectangulo
```

### `main.rs`

```rust
mod matematicas;   // Rust buscará src/matematicas.rs
mod geometria;     // Rust buscará src/geometria/mod.rs

use matematicas::sumar;

fn main() {
    println!("{}", sumar(2, 3));
}
```

### `src/matematicas.rs`

```rust
pub fn sumar(a: i32, b: i32) -> i32 {
    a + b
}
```

### `src/geometria/mod.rs`

```rust
pub mod circulo;
pub mod rectangulo;
```

## Rutas absolutas y relativas

```rust
mod a {
    pub mod b {
        pub fn funcion() {
            println!("en a::b::funcion");
        }
    }
}

fn main() {
    // Ruta absoluta (desde la raíz del crate)
    crate::a::b::funcion();

    // Con use
    use crate::a::b::funcion;
    funcion();
}
```

Dentro de un módulo, puedes usar `super::` para subir un nivel:

```rust
mod padre {
    fn funcion_padre() {}

    mod hijo {
        fn funcion_hijo() {
            super::funcion_padre(); // accede al padre
        }
    }
}
```

## Re-exportar con `pub use`

Para simplificar la API pública de tu biblioteca:

```rust
mod interno {
    pub struct Tipo { /* ... */ }
}

// Re-exporta Tipo en el nivel raíz
pub use interno::Tipo;

// Los usuarios pueden hacer: use mi_crate::Tipo;
// en lugar de: use mi_crate::interno::Tipo;
```

## Estructura limpia recomendada

```
mi_proyecto/
├── Cargo.toml
└── src/
    ├── main.rs         ← punto de entrada, orquesta los módulos
    ├── lib.rs          ← API pública de la librería (opcional)
    ├── modelos.rs      ← structs y enums de datos
    ├── operaciones.rs  ← lógica de negocio
    └── utils.rs        ← utilidades comunes
```

> **Regla de oro:** mantén `main.rs` pequeño — que solo inicialice la aplicación y llame a módulos bien organizados. La lógica vive en módulos separados.
