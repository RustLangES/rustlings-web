# Ejercicio: Ownership y Move

El objetivo es practicar cómo Rust transfiere la propiedad (*ownership*) de los valores.

Cada función **toma ownership** de sus argumentos — después de llamarlas, el valor original queda consumido (movido). Para conservar acceso al dato original, tienes dos opciones: retornar el ownership o usar `.clone()`.

## Funciones a implementar

```rust
// Toma ownership del String y retorna su versión en mayúsculas
fn en_mayusculas(texto: String) -> String

// Toma ownership de ambos strings y retorna su concatenación
fn concatenar(a: String, b: String) -> String

// Retorna la longitud y devuelve el ownership del String
fn longitud(texto: String) -> (usize, String)

// Retorna el original y una copia en mayúsculas — sin consumir el original
fn clonar_upper(texto: String) -> (String, String)
```

## Ejemplos

```
en_mayusculas("hola rust")  → "HOLA RUST"

concatenar("hola", " mundo") → "hola mundo"

longitud("rustlings") → (9, "rustlings")

clonar_upper("hola") → ("hola", "HOLA")
```

## Pistas

- `en_mayusculas`: usa el método `.to_uppercase()` de `String`.
- `concatenar`: usa `format!("{}{}", a, b)` para unir sin preocuparte del borrow.
- `longitud`: primero obtén `texto.len()` (no consume), luego retorna el par `(len, texto)`.
- `clonar_upper`: usa `.clone()` para obtener una copia, luego llama `.to_uppercase()` sobre la copia.
