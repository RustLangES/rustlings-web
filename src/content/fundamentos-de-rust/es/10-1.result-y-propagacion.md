# Result, unwrap y el operador `?`

## Manejo explícito con `match`

La forma más explícita de manejar un `Result`:

```rust
use std::num::ParseIntError;

fn parsear(s: &str) -> Result<i32, ParseIntError> {
    s.trim().parse::<i32>()
}

fn main() {
    match parsear("42") {
        Ok(n)  => println!("Número: {n}"),
        Err(e) => println!("Error: {e}"),
    }
}
```

## `unwrap` y `expect`

Para prototipado rápido o cuando estás seguro de que no hay error:

```rust
fn main() {
    // unwrap: panic si es Err
    let n: i32 = "42".parse().unwrap();

    // expect: panic con mensaje personalizado
    let m: i32 = "42".parse().expect("Se esperaba un número entero");

    println!("{n} {m}");
}
```

**Cuándo usar:**
- `unwrap` / `expect`: en tests, ejemplos y casos donde el error es imposible.
- Nunca en código de producción crítico — propaga el error en su lugar.

## El operador `?`

`?` es azúcar sintáctico para propagación de errores. Equivale a:

```rust
let valor = match resultado {
    Ok(v)  => v,
    Err(e) => return Err(e.into()),
};
```

Úsalo para escribir código lineal sin anidamiento excesivo:

```rust
use std::fs::File;
use std::io::{self, Read};

fn leer_archivo(ruta: &str) -> Result<String, io::Error> {
    let mut archivo = File::open(ruta)?;       // propaga si falla
    let mut contenido = String::new();
    archivo.read_to_string(&mut contenido)?;   // propaga si falla
    Ok(contenido)
}
```

Solo funciona dentro de funciones que retornan `Result` (o `Option`).

## Errores personalizados

Define tu propio tipo de error con un enum:

```rust
use std::fmt;

#[derive(Debug)]
enum ErrorValidacion {
    EdadInvalida(i32),
    NombreVacio,
    EmailInvalido(String),
}

impl fmt::Display for ErrorValidacion {
    fn fmt(&self, f: &mut fmt::Formatter) -> fmt::Result {
        match self {
            ErrorValidacion::EdadInvalida(e) => write!(f, "edad inválida: {e}"),
            ErrorValidacion::NombreVacio     => write!(f, "el nombre no puede estar vacío"),
            ErrorValidacion::EmailInvalido(m) => write!(f, "email inválido: {m}"),
        }
    }
}

fn validar_edad(edad: i32) -> Result<(), ErrorValidacion> {
    if edad < 0 || edad > 150 {
        Err(ErrorValidacion::EdadInvalida(edad))
    } else {
        Ok(())
    }
}

fn main() {
    match validar_edad(200) {
        Ok(())  => println!("Edad válida"),
        Err(e)  => println!("Error: {e}"),
    }
}
```

## Transformar errores con `map_err`

Cuando necesitas convertir el tipo de error:

```rust
fn parsear_edad(s: &str) -> Result<u32, String> {
    s.trim()
     .parse::<u32>()
     .map_err(|_| format!("'{s}' no es una edad válida"))
}
```

## `panic!` — errores irrecuperables

Usa `panic!` solo cuando el programa ha llegado a un estado irrecuperable:

```rust
fn dividir(a: i32, b: i32) -> i32 {
    if b == 0 {
        panic!("División por cero — esto no debería ocurrir");
    }
    a / b
}
```

> **Regla general:** usa `Result` para errores que el caller puede manejar. Usa `panic!` para bugs del programador que nunca deberían ocurrir en código correcto.
