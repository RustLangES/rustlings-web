# Ejercicio: Módulos y Visibilidad

Implementa el contenido del módulo `conversion` usando `pub` para controlar qué es visible desde fuera.

## Qué debes implementar

Dentro del módulo `conversion`:

```rust
pub struct Temperatura {
    pub celsius: f64,
}

impl Temperatura {
    pub fn nueva(celsius: f64) -> Temperatura
    pub fn a_fahrenheit(&self) -> f64   // C × 9/5 + 32
    pub fn a_kelvin(&self) -> f64       // C + 273.15
    pub fn es_congelamiento(&self) -> bool  // true si celsius <= 0.0
}

pub fn celsius_a_fahrenheit(c: f64) -> f64
```

## Conceptos practicados

- **`mod`** — encapsula el código en un módulo.
- **`pub`** — controla qué es accesible desde fuera del módulo (struct, campos, métodos, funciones).
- **`use`** — importa `Temperatura` al ámbito actual para usarlo sin el prefijo `conversion::`.
- **Acceso con prefijo** — `conversion::celsius_a_fahrenheit(...)` accede a una función pública sin `use`.

## Pistas

- El `use conversion::Temperatura` ya está escrito — eso permite usar `Temperatura` directamente en `main` y en los tests.
- Sin `pub`, los campos y métodos no son accesibles desde fuera del módulo.
- Fórmulas: Fahrenheit = `celsius * 9.0 / 5.0 + 32.0`, Kelvin = `celsius + 273.15`.
