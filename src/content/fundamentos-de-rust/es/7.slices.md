# Slices: Referencias a Porciones

Un slice es una **referencia a una porción contigua** de una colección — sin copiar los datos. Los slices tienen dos partes: un puntero al primer elemento y una longitud.

## String slices (`&str`)

Un `&str` es un slice de string — una vista de una porción de texto:

```rust
fn main() {
    let s = String::from("hola mundo");

    let hola  = &s[0..4];   // "hola"
    let mundo = &s[5..10];  // "mundo"
    let todo  = &s[..];     // "hola mundo"

    println!("{hola} | {mundo} | {todo}");
}
```

Los literales de texto como `"hola"` son directamente `&str` — slices que apuntan a datos en el binario del programa.

## Por qué `&str` es mejor que `&String` en parámetros

```rust
// Acepta solo &String
fn longitud_v1(s: &String) -> usize { s.len() }

// Acepta &String, &str, literales — más flexible
fn longitud_v2(s: &str) -> usize { s.len() }

fn main() {
    let s = String::from("hola");

    longitud_v1(&s);        // OK
    // longitud_v1("hola"); // ERROR: tipo incorrecto

    longitud_v2(&s);        // OK (&String coerces to &str)
    longitud_v2("hola");    // OK (literal ya es &str)
}
```

## Array slices (`&[T]`)

Los slices funcionan igual con arrays y vectores:

```rust
fn main() {
    let a = [1, 2, 3, 4, 5];

    let slice = &a[1..3]; // [2, 3]
    println!("{:?}", slice);
    println!("longitud: {}", slice.len());

    // Función genérica sobre slices
    imprimir_slice(&a);
    imprimir_slice(&a[2..]);
}

fn imprimir_slice(s: &[i32]) {
    for x in s {
        print!("{x} ");
    }
    println!();
}
```

## Slices y ownership

Los slices son referencias — no son dueños de los datos:

```rust
fn main() {
    let v = vec![10, 20, 30, 40, 50];

    let slice = &v[1..4]; // referencia inmutable a v

    // v.push(60); // ERROR: no puedes modificar v mientras el slice existe
    println!("{:?}", slice);
    // Aquí termina el uso del slice, ahora sí podrías modificar v
}
```

## Operaciones útiles con slices

```rust
fn main() {
    let numeros = [3, 1, 4, 1, 5, 9, 2, 6];
    let slice = &numeros[..];

    println!("longitud: {}", slice.len());
    println!("primer: {:?}", slice.first());
    println!("último: {:?}", slice.last());
    println!("contiene 5: {}", slice.contains(&5));

    // Iteración
    let suma: i32 = slice.iter().sum();
    let max = slice.iter().max().unwrap();
    println!("suma={suma}, max={max}");
}
```

## Reglas de vida implícitas (lifetime elision)

Cuando una función toma un slice y retorna uno, Rust infiere que el slice retornado apunta a los datos del parámetro:

```rust
// El lifetime es implícito: la & retornada apunta a los datos de `s`
fn primera_palabra(s: &str) -> &str {
    match s.find(' ') {
        Some(i) => &s[..i],
        None => s,
    }
}

fn main() {
    let texto = String::from("hola mundo");
    let palabra = primera_palabra(&texto);
    println!("{palabra}"); // "hola"
    // texto no puede modificarse aquí porque `palabra` lo está referenciando
}
```

> **Resumen:** `&[T]` y `&str` son los slices de arrays y strings respectivamente. Úsalos en funciones en lugar de referencias a colecciones completas — son más flexibles y más idiomáticos en Rust.
