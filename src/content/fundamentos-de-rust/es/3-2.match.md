# match

`match` es la herramienta de *pattern matching* de Rust. Es como un `switch` con superpoderes: es **exhaustivo** (debes cubrir todos los casos) y puede extraer valores de estructuras complejas.

## match básico

```rust
fn main() {
    let valor = 4;

    match valor {
        1 => println!("Uno"),
        2 => println!("Dos"),
        3 => println!("Tres"),
        _ => println!("Otro"),  // _ es el comodín (catch-all)
    }
}
```

Si olvidas el comodín `_` y no cubres todos los posibles valores, el compilador falla:

```
error[E0004]: non-exhaustive patterns
```

## Patrones múltiples y rangos

```rust
fn main() {
    let c = 'b';

    match c {
        'a' | 'e' | 'i' | 'o' | 'u' => println!("vocal"),
        'a'..='z' => println!("consonante minúscula"),
        'A'..='Z' => println!("consonante mayúscula"),
        _ => println!("otro carácter"),
    }
}
```

## match como expresión

Al igual que `if`, `match` retorna un valor:

```rust
fn main() {
    let numero = 7;

    let tipo = match numero {
        n if n < 0 => "negativo",
        0 => "cero",
        n if n % 2 == 0 => "par positivo",
        _ => "impar positivo",
    };

    println!("{numero} es {tipo}");
}
```

Las condiciones extra (`if n < 0`) se llaman **guards**.

## Destructuring en match

`match` puede descomponer tuplas, structs y enums:

```rust
fn main() {
    let punto = (3, -2);

    match punto {
        (0, 0) => println!("Origen"),
        (x, 0) => println!("Sobre el eje X: {x}"),
        (0, y) => println!("Sobre el eje Y: {y}"),
        (x, y) => println!("Punto ({x}, {y})"),
    }
}
```

## Capturar el valor con `@`

```rust
fn main() {
    let numero = 5;

    match numero {
        n @ 1..=9 => println!("Dígito: {n}"),
        n @ 10..=99 => println!("Dos dígitos: {n}"),
        _ => println!("Muy grande"),
    }
}
```

> **Clave:** `match` es exhaustivo por diseño. El compilador te garantiza que nunca te olvidaste de un caso.
